# Security and information handling

The local CLI reads the files you point it to and makes no network calls. It does not execute user-supplied code or automatically transmit decision briefs.

The repository also contains prompts that users may copy into external AI services. Those services have separate retention, access, training, residency, and contractual terms.

- Use fictional, public, or explicitly authorized material.
- Do not paste secrets, credentials, personal data, confidential client/employer material, privileged advice, or restricted records into an external model.
- De-identification or summarization alone is not authorization; consider re-identification and contractual restrictions.
- Treat AI output as a draft challenge or transformation, never as source evidence or independent review.
- Review generated reports before circulation and publish only permission-cleared artifacts.

Report code vulnerabilities through the repository's security-reporting channel without including live secrets or sensitive source material.
