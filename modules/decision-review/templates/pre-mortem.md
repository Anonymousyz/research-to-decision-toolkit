# Decision pre-mortem

Assume the decision was implemented and failed badly. Work backward without defending the proposal.

1. Generate at least five distinct causes across workflow, evidence, incentives, security/operations, and stakeholder adoption.
2. Name one sleeper risk that is easy to ignore because it is politically or professionally uncomfortable.
3. For each cause, record an early signal, owner, mitigation, and stop condition.

| Failure cause | Earliest signal | Owner | Preventive action | Stop condition |
|---|---|---|---|---|
| | | | | |

Run individual generation before group discussion to reduce anchoring. Consolidate duplicates only after every participant has contributed.
